01 — contexto

O problema
real do briefing.

Toda social media já ouviu alguma versão de "pode fazer o conteúdo do jeito que achar melhor" — e sabe o quanto isso complica, não facilita. O problema não é falta de boa vontade do cliente: é que perguntas abertas mal formuladas não extraem as informações que realmente importam.

O briefing padrão de mercado coleta dados superficiais. Esta ferramenta foi criada para coletar o que o cliente sabe mas não sabe que sabe — os diferenciais reais, as objeções que enfrenta, o que nunca quer ser comparado. E depois transformar tudo isso em diagnóstico estratégico real, não em resumo das respostas.

06 — decisões de design

O porquê de cada
escolha.

Sem login para o cliente

Cada etapa de fricção no formulário é uma oportunidade de abandono. Um link direto que abre e funciona — sem conta, sem senha — respeita o tempo do cliente e aumenta a taxa de resposta completa.

Análise na linguagem do cliente, não do mercado

O prompt para a API foi construído para gerar hooks e posicionamentos usando as palavras exatas que o cliente usou no briefing. A melhor copy é a que o próprio cliente escreveu sem saber.

Dashboard oculto, não separado

Separar cliente e profissional em dois URLs diferentes cria problema de manutenção. Ocultar via código no mesmo link mantém a experiência do cliente limpa enquanto o dashboard existe no mesmo arquivo.

Perguntas com contexto, não só campo vazio

Cada pergunta tem uma dica sobre o que está sendo pedido — não para facilitar demais, mas para ativar o pensamento certo. "O que você faz que nenhum concorrente faz igual..." direciona mais que "qual o diferencial?"

08 — aprendizados

✦ o que esse projeto ensinou

Além do formulário.

  • Perguntas mal formuladas custam horas de retrabalho. O investimento em construir o briefing certo é menor do que o custo de refazer conteúdo por falta de contexto.
  • O cliente não sabe o que sabe. Perguntas que pedem comparação ("o que você NUNCA quer ser") extraem mais do que perguntas diretas sobre atributos positivos.
  • A IA não cria posicionamento — ela revela o que já estava nas respostas. O prompt certo transforma dados brutos em linguagem estratégica operável.
  • Fricção mata adoção. Cada campo a mais, cada login, cada etapa desnecessária é uma desistência que acontece antes de você nem saber.
