01 — contexto

O problema com
os números.

Ver os dados do Instagram sem saber o que eles significam estrategicamente é o equivalente a ler uma análise de sangue sem médico — você sabe que tem números, mas não sabe o que fazer com eles.

As ferramentas nativas da plataforma mostram o dado, não o diagnóstico. Alcance alto com engajamento baixo é um problema diferente de alcance baixo com engajamento alto — e cada um pede uma ação diferente. Essa distinção raramente aparece no painel do Instagram.

O problema adicional: a maioria das pessoas acessa os dados via print, não via exportação de CSV. Construir para o comportamento real, não para o ideal.

02 — como funciona

Do print ao
diagnóstico.

01

Upload dos prints

Você faz upload dos screenshots das métricas do Instagram — alcance, impressões, interações, perfis alcançados. Sem precisar exportar nada, sem CSV, sem configuração.

02

Contexto do perfil

Você informa o objetivo principal do perfil (autoridade, vendas, comunidade, engajamento) e o segmento. Isso calibra o diagnóstico — um perfil de e-commerce tem métricas saudáveis diferentes de um perfil pessoal.

03

Leitura das imagens pela IA

A Claude API usa visão computacional para ler os números diretamente dos prints — sem OCR separado, sem pré-processamento manual. Identifica os dados relevantes e descarta ruído visual.

04

Três blocos de output

Leitura dos dados — o que os números dizem, em português claro. Diagnóstico estratégico — o que está funcionando, o que não está e por quê. Ações prioritárias — o que fazer primeiro, com razão para isso.

03 — exemplo de output

O que a ferramenta
entrega.

✦ análise gerada — perfil de mentoria
✦ leitura dos dados

Alcance de 4.230 perfis únicos com 312 interações — taxa de engajamento de 7,4%, acima da média de mercado para perfis de serviço (3–5%). O dado crítico: 68% do alcance veio de não seguidores, o que indica forte distribuição orgânica via Explorar ou compartilhamento.

✦ diagnóstico estratégico

A peça está atraindo novos públicos mas a conversão em seguidores não está acontecendo — o conteúdo atrai, mas não retém. O problema provavelmente está no CTA ou na expectativa criada vs. o que o perfil entrega após o primeiro clique.

✦ ações prioritárias

1. Auditar os últimos 3 posts publicados após esta peça — se o engajamento caiu, o conteúdo seguinte quebrou a promessa implícita. 2. Testar uma versão da mesma peça com CTA explícito para seguir. 3. Identificar qual elemento visual gerou o alcance expandido e replicar a estrutura.

04 — decisões de design

O porquê de cada
escolha.

Prints, não exportação

A exportação de dados do Instagram exige permissões, apps terceiros ou acesso ao Creator Studio. O print é o formato natural de como a maioria das pessoas documenta os dados. Projetar para o comportamento real elimina fricção antes de começar.

Contexto do objetivo antes do diagnóstico

Um alcance de 2.000 pessoas pode ser excelente para um perfil local de 500 seguidores e medíocre para um perfil de produto nacional. Sem contexto, qualquer análise é genérica. Pedir o objetivo primeiro calibra o diagnóstico para a realidade específica do perfil.

Três blocos de output com funções distintas

Separar leitura, diagnóstico e ação evita o erro comum de misturar "o que aconteceu" com "o que fazer". São três momentos cognitivos diferentes — juntos numa só resposta, o usuário tende a ler e não conseguir aplicar.

05 — construção

Como foi feita.

A ferramenta surgiu de uma limitação recorrente: receber prints de clientes com métricas e precisar analisá-los manualmente, uma por uma, sem padrão de estrutura. A pergunta foi: o que acontece se eu deixar a IA ler esses prints diretamente?

A resposta foi melhor do que o esperado. A visão computacional da Claude API identifica números em screenshots com alta precisão — inclusive quando há variações de layout entre as diferentes telas do Instagram.

06 — aprendizados

✦ o que esse projeto ensinou

Além dos
números.

  • Dados sem contexto são só números. A mesma métrica tem significados completamente diferentes dependendo do objetivo, do tamanho da base e do momento do perfil.
  • Visão computacional muda o que é possível. Conseguir ler prints diretamente abre uma classe inteira de ferramentas que antes exigiam OCR separado, exportações manuais ou APIs pagas de terceiros.
  • O diagnóstico é mais valioso que a leitura. Qualquer um consegue ver que o alcance foi 4.000. O que a maioria não sabe é o que isso significa para o perfil específico.
  • Projetar para o comportamento real, não o ideal. A decisão de aceitar prints em vez de exigir exportação foi a decisão que tornou a ferramenta utilizável de verdade.
