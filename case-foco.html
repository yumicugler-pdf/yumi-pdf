01 — contexto

Paralisia não é
preguiça.

TDAH não é déficit de atenção — é déficit de regulação da atenção. A atenção existe em abundância para coisas que capturam o cérebro. O problema é iniciar tarefas abertas, sem estrutura, sem urgência clara, sem um primeiro passo óbvio e minúsculo.

As ferramentas de produtividade convencionais foram projetadas para cérebros neurotípicos. Listas longas, deadlines abstratos, sistemas de prioridade com muitas variáveis. Para o TDAH, quanto mais complexo o sistema, maior a probabilidade de abandono antes de começar.

A pergunta que gerou essa ferramenta foi: qual é o mínimo possível de estrutura para que o primeiro passo aconteça?

03 — princípios de design

Projetado para como
o TDAH funciona.

Princípio 01
Um passo por vez

Ver todos os passos de uma vez recria a sobrecarga que causa a paralisia. A ferramenta mostra só o passo atual — os demais existem mas ficam em segundo plano visual.

Princípio 02
Primeiro passo ridículo

O primeiro passo gerado é sempre absurdamente fácil — "abrir a pasta", "abrir o documento". A inércia é o maior obstáculo. Quebrar a inércia com um passo que não dá para falhar.

Princípio 03
Timer como urgência artificial

O cérebro com TDAH responde bem a urgência real. O timer não pune — ele cria a sensação de limite que ativa o foco. Sem timer, o passo pode durar indefinidamente.

Princípio 04
Estética escura, intencional

Interface escura reduz distração visual. O contraste alto entre o texto do passo ativo e o fundo cria um ponto focal claro — o olho vai para onde precisa ir sem esforço.

04 — decisões de design

O porquê de cada
escolha.

Máximo de 6 passos — não mais

Mais de 6 passos recria a sensação de tarefa longa e retoma a sobrecarga. 6 é o limite onde a tarefa ainda parece gerenciável visualmente, mesmo que leve horas no total. A restrição é o design.

Input livre, sem formulário

"Escrever pasta do projeto Holmy e verificar as métricas do Instagram para o relatório de mai" gera passos tão bons quanto um input bem estruturado. Pedir formatação antes de começar é fricção que pode causar desistência antes mesmo de iniciar.

Progresso visível, não numérico

Mostrar "passo 2 de 6" ativa a parte do cérebro que calcula quanto falta. A barra de progresso visual é percebida como movimento sem ativar a mesma sobrecarga cognitiva do número explícito.

05 — construção

Como foi feita.

A conversa que gerou essa ferramenta começou com um compartilhamento honesto sobre como o TDAH afeta a execução. Em vez de sugestões genéricas de produtividade, o processo de co-design foi perguntar: onde trava exatamente? O que paralisa mais? Como você processa instruções?

Cada resposta virou um princípio de design. O diagnóstico virou especificação. A ferramenta não foi construída para um usuário genérico com dificuldade de foco — foi construída para um padrão específico e documentado de como o TDAH se manifesta no trabalho criativo.

06 — aprendizados

✦ o que esse projeto ensinou

Além do
foco.

  • Limitações bem diagnosticadas viram especificações de design. TDAH não é obstáculo para o projeto — é dado de entrada. Saber como o cérebro funciona permitiu criar algo que realmente funciona para ele.
  • A restrição é o produto. Limitar a 6 passos, mostrar um por vez, começar pelo ridiculamente fácil — cada limitação foi uma decisão de design intencional, não uma simplificação preguiçosa.
  • Urgência artificial funciona. O timer não está medindo desempenho — está criando a estrutura temporal que o cérebro com TDAH precisa para iniciar e manter o movimento.
  • Projetar para si mesma é diferente de projetar para uma persona. Cada decisão foi testada em condições reais — dias de hiperfoco e dias de paralisia total. Isso muda o que sobrevive ao protótipo.
